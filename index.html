http-pulse

What It http-pulse?

http-pulse is a cli program for configuring http monitors on pulse.atmos.org, a service I wrote to learn a little more about node.js and mongohq. It's basically a free HTTP monitoring service, with a REST API.

I'm hoping to find the time to build an iPhone UI on top of this service as well, but for now there's only a command line interface.

Getting Started

The short and sweet description of http-pulse is that it's a silly crud app. That means you can do the following things from the CLI.

  • Add a new monitor
  • Remove an existing monitor
  • Check the status of all monitors

http-pulse will happily check these monitors every minute and expose the latest status to you via an HTTP API

Installing the CLI tool

You'll need to install the http-pulse gem: 

% gem install http-pulse
Successfully installed http-pulse-0.0.4
1 gem installed

Signing Up

The first thing you'll need to do is authenticate to get an API token. Visit the pulse app and login with github.

After you successfully login you should be greeted by a page like this:

Registering Yourself

At that point you can register yourself via the command line. 

% http-pulse --register 1e8e4b3a31f860ac43e30e0ed15eb1eebac49b1b
Added your token, now try running 'http-pulse' to see your monitors

Viewing Your Monitors

You simply run 'http-pulse' to see your monitor's status. 

% http-pulse
Your Monitors
-------------------------------------------------------------
http://www.atmos.org          UP as of 2010/08/23 19:32 PDT
http://github.com/atmos       UP as of 2010/08/23 19:32 PDT

Adding a New Monitor

% http-pulse --create http://github.com/atmos
Successfully added http://github.com/atmos

Removing a Monitor

% http-pulse --delete http://www.atmos.org
Successfully removed http://www.atmos.org

Wait.. Why?

With the node knockout a few days away, I figured it was a good time to build another small application on node.js. I knew I wanted to host on heroku and figured I should take the time to see what types of technologies I could take advantage of. I was most interested in their add-ons which give me access to services I wouldn't want to setup/monitor during the node knockout competition. Hopefully this example will help some people out.
